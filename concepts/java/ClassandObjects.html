Java Classes

A class is a blueprint from which individual objects are created. It contains fields (variables) and methods.

Key Futures of a Class:

1.Fields:Variables that holds the state of the objects.

2.Methods:Functions that define the behavior of the object.

3.Constructors:Special methods used to initialize objects.

4.Access Modifiers:Control the visibility of the class and its members (e.g., public, private, protected).

Components of Java

In general, class declarations can include these components, in order:

  • Modifiers:A class can be public or has default access (Refer this for details).
  • Class keyword:Class keyword is used to create a class.
  • Class name:The name should begin with an initial letter (capitalized by convention).
  • Superclass (if any):The name of the class's parent (superclass), if any, preceded by the keyword extends. A class can only extend (subclass) one parent.
  • Interfaces(if any):A comma-separated list of interfaces implemented by the class, if any, preceded by the keyword implements. A class can implement more than one interface.
  • Body:The class body is surrounded by braces, { }.

    • Constructors are used for initializing new objects. Fields are variables that provide the state of the class and its objects, and methods are used to implement the behavior of the class and its objects. There are various types of classes that are used in real-time applications such as nested classes, anonymous classes and lambda expressions.

    Objects in Java

    In Java, objects are the fundamental building blocks of object-oriented programming. They represent real-world entities and are instances of classes. Here's a concise overview:

    What is an Object?

    An object is a runtime entity that:
  • Has State:Represented by fields (variables).
  • Has Behavior:Represented by methods (functions).
  • Has Identity:A unique reference in memory.

    • For example, a Car object might have:

  • State:Color, brand, speed.
  • Behavior:Start, stop, accelerate.

    • Example 
    // Example of object creation
class Car {
    String color;
    int speed;

    void start() {
        System.out.println("Car started");
    }
}
public class Main {
    public static void main(String[] args) {
        Car myCar = new Car(); // Object creation
        myCar.color = "Red";   // Setting state
        myCar.start();         // Calling behavior
    }
}

    Difference Between

    Class Objects
    Class is the blueprint of an object. It is used to create objects. An object is an instance of the class.
    No memory is allocated when a class is declared. Memory is allocated as soon as an object is created.
    A class is a group of similar objects. An object is a real-world entity such as a book, car, etc.
    Class is a logical entity. An object is a physical entity.
    A class can only be declared once. Objects can be created many times as per the requirement.
    An example of class can be a car. Objects of the class car can be BMW, Mercedes, Ferrari, etc.

    Java Access Modifiers

    In Java, access modifiers are keywords used to set the visibility and accessibility of classes, methods, variables, and constructors. They help enforce encapsulation and control how different parts of a program interact. There are four main access modifiers:

    1. Public

    Scope: Accessible from anywhere in the program, across all packages.
    Usage: Typically used for classes, methods, or variables that need to be universally accessible.

    public class MyClass {
    public void display() {
        System.out.println("This is a public method.");
    }
}

    2. Private

    Scope: Accessible only within the same class.
    Usage: Used to restrict access to sensitive data or methods, ensuring encapsulation.

    public class MyClass {
    private int secretNumber = 42;

    private void showSecret() {
        System.out.println("Secret: " + secretNumber);
    }
}

    3. Protected

    Scope: Accessible within the same package and by subclasses (even in different packages).
    Usage: Often used in inheritance to allow controlled access to members by child classes.

    public class Parent {
    protected void greet() {
        System.out.println("Hello from the parent class!");
    }
}

public class Child extends Parent {
    public void callGreet() {
        greet(); // Accessible due to 'protected'
    }
}

    4. Default (Package-Private)

    Scope: Accessible only within the same package (no keyword is used).
    Usage: Useful for package-level encapsulation.

    
class MyClass {
void display() {
System.out.println("This is a default method.");
    }
}
